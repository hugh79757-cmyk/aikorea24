바이브코딩을 시작하려면 가장 먼저 결정해야 할 것이 있습니다. **어떤 AI 도구를 쓸 것인가** 입니다. 현재 바이브코딩에 활용할 수 있는 AI 도구는 꽤 많습니다. ChatGPT, Claude, Cursor, GitHub Copilot, Claude Code 등이 대표적입니다. 이름만 들어도 머리가 복잡해집니다. 이 강의에서는 모든 과정은 대화형 AI를 통해서만 진행된다는 점, 참고하세요. 클로드 웹버전과 vs code 프로그램을 사용하고 있습니다. 클로드 웹버전의 경우 [젠스파크](https://www.genspark.ai/)를 통해 구매를 했고, [vs code](https://aikorea24.kr/blog/nae-keompyuteoe-gaebalhwangyeong-mandeulgi/)는 무료입니다. 



완전 초보라면 **ChatGPT 또는 Claude 웹 버전으로 시작하고, 프로젝트가 커지면 Cursor로 넘어가는 것** 이 가장 현실적인 경로입니다. 각 도구가 왜 존재하고, 어떤 상황에 맞는지 하나씩 짚어보겠습니다.

## 도구를 고르기 전에 알아야 할 것

AI 코딩 도구는 크게 두 종류로 나뉩니다.

첫 번째는 **대화형 도구** 입니다. 웹 브라우저에서 AI와 채팅하듯 대화하면서 코드를 받는 방식입니다. ChatGPT와 Claude가 여기에 해당합니다. "자기소개 웹페이지 만들어줘"라고 말하면 AI가 코드를 생성해주고, 그 코드를 복사해서 파일에 붙여넣는 식입니다. 카카오톡으로 대화하는 것처럼 편하지만, 코드를 직접 파일로 옮기는 수작업이 필요합니다.

두 번째는 **에디터 통합형 도구** 입니다. 코드를 작성하는 편집기(에디터) 안에 AI가 내장되어 있어서, 코드를 쓰는 도중에 바로 AI의 도움을 받을 수 있습니다. Cursor와 GitHub Copilot이 여기에 해당합니다. 코드를 복사 붙여넣기 할 필요 없이 에디터 안에서 모든 것이 해결됩니다.

어떤 종류가 좋다 나쁘다의 문제가 아닙니다. **지금 나에게 맞는 것** 이 무엇인지가 중요합니다.

## ChatGPT 가장 익숙하고 범용적인 도구

ChatGPT는 OpenAI가 개발한 대화형 AI입니다. 코딩뿐 아니라 글쓰기, 번역, 분석 등 거의 모든 작업에 활용할 수 있습니다. 웹 브라우저에서 바로 사용할 수 있어 접근성이 가장 좋고, 이미 많은 분들이 일상에서 사용하고 있을 겁니다.

바이브코딩에서의 강점은 **대화 맥락을 잘 기억한다는 점** 입니다. "아까 만든 코드에서 배경색을 파란색으로 바꿔줘"처럼 이전 대화를 참조하는 요청이 자연스럽게 됩니다. 코드 외에 "이 에러가 무슨 뜻이야?"라고 물어보면 초보자 눈높이에서 설명해주기 때문에 학습 도구로도 우수합니다.

단점도 있습니다. 코드가 길어지면 중간에 잘리는 경우가 있고, 생성된 코드를 에디터에 수동으로 복사해서 붙여넣어야 합니다. 프로젝트 파일이 여러 개로 늘어나면 어떤 파일에 어떤 코드를 넣어야 하는지 관리하기가 번거로워집니다.

무료 버전으로도 바이브코딩을 시작할 수 있습니다. 다만 사용량에 제한이 있어서 본격적으로 프로젝트를 진행하려면 월 20달러의 Plus 요금제가 필요합니다.

## Claude 긴 코드와 정교한 작업에 강한 도구

[2026년 3월 7일 (토) AI 브리핑 - AI코리아24](https://aikorea24.kr/briefing/2026-03-07/)

엔트로픽과 오픈AI의 성격은 최근 팬타곤이 엔트로픽과의 계약을 해지하고 오픈AI와 비밀 계약을 했던 것으로 뚜렷하게 구분이 됩니다. 자세한 내용은 위에 뉴스브리핑을 통해 확인해보세요.



Claude는 Anthropic이 개발한 AI입니다. ChatGPT와 마찬가지로 웹 브라우저에서 대화형으로 사용합니다. Claude의 가장 큰 특징은 **긴 문맥을 처리하는 능력** 입니다. 한 번에 수백 페이지 분량의 코드나 문서를 읽고 분석할 수 있습니다.

![img_20260310_143710.webp](https://img.aikorea24.kr/images/바이브코딩-워크쓰루-0-1/cf98e8eef34309e7d39cb97ad4087c46c4c09312.webp)

바이브코딩에서 이것이 왜 중요하냐면, 프로젝트가 조금만 커져도 파일이 여러 개 생기고 코드가 길어집니다. 이때 기존 코드 전체를 AI에게 보여주면서 "여기에 검색 기능 추가해줘"라고 요청할 수 있어야 합니다. Claude는 이런 작업에서 뛰어난 성능을 보여줍니다.

코드 설명이 상세하고 구조적입니다. "이 코드가 뭘 하는 건지 모르겠어"라고 하면 함수 단위로 나눠서 차근차근 설명해줍니다. 학습 목적으로도 ChatGPT 못지않게 유용합니다.

단점은 무료 버전의 사용량 제한이 ChatGPT보다 빡빡하다는 점입니다. 본격적으로 활용하려면 월 20달러의 Pro 요금제를 권장합니다.

최근에는 **Claude Code** 라는 터미널 기반 도구도 출시되었습니다. 터미널(컴퓨터에 텍스트 명령어를 입력하는 검은 화면)에서 직접 Claude와 대화하면서 코드를 생성하고, 파일을 만들고, 수정까지 자동으로 해줍니다. 웹 브라우저와 에디터를 오가며 복사 붙여넣기를 할 필요가 없어서 작업 효율이 크게 올라갑니다. 다만 터미널 사용이 아직 낯선 입문자에게는 진입장벽이 있으므로, 이 시리즈의 선수과목을 마친 후에 도전하시는 것을 권합니다.

## Cursor 에디터 안에서 AI와 함께 코딩하는 도구

Cursor는 VS Code를 기반으로 만들어진 **AI 전용 코드 에디터** 입니다. 에디터 안에서 바로 AI에게 질문하고, 코드를 생성하고, 수정할 수 있습니다. 대화형 도구들처럼 별도의 웹사이트를 열 필요가 없습니다. 코드를 쓰는 곳에서 바로 AI의 도움을 받습니다.

가장 큰 장점은 **프로젝트 전체 맥락을 이해한다는 것** 입니다. ChatGPT나 Claude에게 코드를 요청하면 해당 대화에서 주고받은 내용만 참고합니다. 하지만 Cursor는 프로젝트 폴더에 있는 모든 파일을 읽고, 그 맥락 위에서 코드를 생성합니다. "이 프로젝트의 스타일에 맞춰서 새 페이지 만들어줘"라는 요청이 가능합니다.

코드를 선택한 후 "이 부분 설명해줘", "여기 버그 수정해줘"처럼 특정 코드를 지목해서 AI와 대화할 수 있습니다. 또한 Claude와 GPT 모델을 선택해서 사용할 수 있어, 상황에 따라 최적의 AI 모델을 골라 쓸 수 있습니다.

단점은 VS Code를 사용해본 적 없는 분에게는 에디터 자체가 낯설 수 있다는 점입니다. 또한 무료 버전의 AI 사용 횟수가 제한되어 있어서, 본격적으로 활용하려면 월 20달러의 Pro 요금제가 필요합니다.

## GitHub Copilot 코드 자동완성의 전문가

GitHub Copilot은 GitHub과 OpenAI가 협력하여 만든 AI 코딩 어시스턴트입니다. VS Code, JetBrains 등 기존에 쓰던 에디터에 플러그인(확장 프로그램) 형태로 설치합니다. Cursor처럼 에디터 자체를 바꾸는 것이 아니라, 기존 에디터에 AI 기능을 추가하는 방식입니다.

가장 큰 특징은 **실시간 자동완성** 입니다. 코드를 쓰다 보면 AI가 다음에 올 코드를 예측해서 회색 글씨로 제안합니다. Tab 키를 누르면 제안을 수락하고, 마음에 안 들면 그냥 계속 타이핑하면 됩니다. 반복적인 코드를 작성할 때 생산성이 크게 올라갑니다.

다만 Copilot은 **이미 코딩을 어느 정도 할 줄 아는 사람** 에게 최적화되어 있습니다. "웹페이지 만들어줘"라고 대화하는 방식이 아니라, 코드를 쓰는 과정에서 보조하는 역할이기 때문입니다. 바이브코딩 입문 단계에서는 우선순위가 낮습니다.

월 10달러(개인)이며, 최근 무료 플랜도 제공하기 시작했습니다.

## 입문자를 위한 현실적인 추천 경로

도구가 많아서 오히려 고민이 깊어질 수 있습니다. 현실적인 경로를 제안합니다.

**1단계** 에서는 ChatGPT 또는 Claude 웹 버전으로 시작합니다. 웹 브라우저만 있으면 바로 쓸 수 있고, 코드뿐 아니라 개념 질문도 할 수 있어서 학습과 실습을 동시에 진행할 수 있습니다. 이 워크쓰루 시리즈의 시리즈1(내 첫 웹사이트)까지는 이 조합으로 충분합니다. 왼쪽 화면에 AI 대화창, 오른쪽 화면에 VS Code를 띄워놓고 복사 붙여넣기로 진행합니다.

**2단계** 에서는 프로젝트가 커지면 Cursor로 전환합니다. 파일이 5개 이상으로 늘어나고, 코드를 복사 붙여넣기하는 것이 번거로워지기 시작하면 그때가 Cursor로 넘어갈 타이밍입니다. 이 시리즈의 시리즈2(공공데이터 API 서비스) 이후가 적절한 시점입니다.

**3단계** 에서는 터미널에 익숙해지면 Claude Code를 병행합니다. 파일 생성, 수정, 빌드, 배포를 대화만으로 처리할 수 있어 작업 속도가 한 단계 올라갑니다.

어떤 도구를 선택하든 중요한 것은 **하나를 먼저 익숙하게 쓰는 것** 입니다. 여러 도구를 동시에 배우려고 하면 어느 것도 제대로 익히지 못합니다. 하나에 익숙해진 후에 다음 도구로 넘어가세요.

## 요금 정리

각 도구의 무료 범위와 유료 요금을 정리합니다.

**ChatGPT** 는 무료 버전이 있고, 고급 모델과 더 많은 사용량이 필요하면 월 20달러의 Plus 요금제를 선택합니다.

**Claude** 는 무료 버전이 있고, 사용량 제한을 해소하려면 월 20달러의 Pro 요금제를 선택합니다.

**Cursor** 는 무료 버전에서 월 2,000회의 자동완성과 50회의 느린 요청이 가능하고, Pro 요금제는 월 20달러입니다.

**GitHub Copilot** 은 무료 플랜이 있고, 개인 유료 플랜은 월 10달러입니다.

**Claude Code** 는 Claude Pro 구독(월 20달러) 안에서 사용할 수 있고, API 사용 시 토큰 기반 종량제입니다.

입문 단계에서는 ChatGPT 또는 Claude 무료 버전만으로도 시작할 수 있습니다. 다만 프로젝트를 진행하다 보면 무료 한도에 금방 도달하므로, 본격적으로 바이브코딩을 할 생각이라면 월 20달러 정도의 투자는 현실적으로 필요합니다.

## 정리

바이브코딩 도구는 많지만 고민할 필요는 없습니다. ChatGPT나 Claude로 시작해서 감을 잡고, 프로젝트가 커지면 Cursor로 전환하면 됩니다. 도구 선택에 시간을 쓰기보다 하나를 골라서 바로 시작하는 것이 가장 좋습니다.
