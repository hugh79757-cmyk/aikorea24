앞의 강좌에서 배포가 완료되어 `my-first-website.pages.dev` (https://lecture-23v.pages.dev/)라는 주소로 웹사이트에 접속할 수 있게 되었습니다. 무료이고 잘 작동합니다. 하지만 이 주소를 명함에 넣기에는 좀 아쉽습니다. `pages.dev`가 붙어있어서 내 주소라는 느낌이 약합니다.

이번 글에서는 **나만의 도메인** 을 구매하고, 배포된 웹사이트에 연결하는 방법을 알아봅니다. 도메인을 연결하면 `honggildong.com`이나 `hong-portfolio.kr` 같은 주소로 접속할 수 있게 됩니다. HTTPS 보안 인증서도 Cloudflare가 자동으로 처리해주므로 별도 설정이 필요 없습니다.

## 도메인이 뭔가요

도메인은 **인터넷 세계의 주소** 입니다. 현실에서 친구 집에 가려면 "서울시 강남구 테헤란로 123"이라는 주소가 필요하듯, 인터넷에서 웹사이트에 가려면 도메인이 필요합니다.

실제로 컴퓨터들은 IP 주소라고 하는 숫자(예: 104.21.54.162)로 서로를 식별합니다. 하지만 사람이 이런 숫자를 기억하기는 어렵습니다. 그래서 숫자 대신 사람이 기억하기 쉬운 이름을 붙인 것이 도메인입니다. `aikorea24.kr`이라고 입력하면 컴퓨터가 자동으로 해당 IP 주소를 찾아서 접속해줍니다.

도메인은 구매하는 것이 아니라 정확히는 **임대** 하는 것입니다. 보통 1년 단위로 사용료를 내고, 갱신하지 않으면 다른 사람이 사용할 수 있게 됩니다. .com 도메인 기준 연 20달러 정도입니다.

## 어디서 도메인을 사나요

도메인을 구입할 수 있는 곳 도메인은 아래 업체들에서 구입할 수 있습니다. 가격은 도메인 종류(.com, .kr, .io 등)에 따라 다르며, .com 기준 연간 2만 원~ 3만 원 사이입니다. 

**추천 (해외, 가성비)** - [Cloudflare ](https://domains.cloudflare.com/) — 원가 그대로 판매하는 것이 특징입니다. 마진을 붙이지 않아서 가장 저렴한 경우가 많습니다. 이 시리즈에서 Cloudflare Pages를 사용하므로 도메인도 여기서 구입하면 연결이 가장 간단합니다. .kr 도메인은 이곳에서 구매할 수 없습니다.

**해외 대형 업체** - [GoDaddy](https://www.godaddy.com/) — 세계에서 가장 큰 도메인 등록 업체입니다. 한국어 지원이 되고 도메인 종류가 가장 많습니다. 다만 갱신 가격이 비싼 편이니 첫해 할인에만 끌리지 않도록 주의하세요. 

**국내 업체 (.kr 도메인에 유리)**

- [가비아](https://domain.gabia.com/) — 국내 도메인 점유율 1위 업체입니다. .kr, .co.kr 같은 한국 도메인을 등록할 때 편리합니다. 한국어 고객지원이 확실합니다. 

- [호스팅케이알](https://www.hosting.kr/) — 26년 운영 경력의 국내 대표 도메인 업체입니다. .kr 도메인 가격이 합리적이고, 도메인과 호스팅을 함께 관리할 수 있습니다. 

- [후이즈](https://whois.co.kr/) — 국내 유일 일반최상위도메인 관리기관 운영사업자입니다. .com, .net 등 국제 도메인과 .kr 도메인 모두 등록할 수 있습니다.

이 시리즈에서는 Cloudflare Pages로 배포하기 때문에, **Cloudflare** 에서 도메인을 구입하면 설정이 가장 간단합니다. 다른 곳에서 구입한 도메인도 Cloudflare에 연결할 수 있지만, 네임서버 변경이라는 추가 단계가 필요합니다.

## 도메인 이름 정하기

도메인 이름을 정하는 것은 생각보다 중요합니다. 한번 정하면 바꾸기 어렵기 때문입니다.

**짧고 기억하기 쉬운 이름** 을 추천합니다. 본인 이름의 영문 표기(honggildong.com), 활동명이나 브랜드명(mycafe.kr), 프로젝트의 성격을 나타내는 이름(seoul-guide.com) 등이 좋습니다.

**확장자(TLD)** 도 선택해야 합니다. .com이 가장 보편적이고, .kr은 한국 도메인으로 국내 사용자에게 신뢰감을 줍니다. .dev는 개발 관련 프로젝트에 어울립니다. 가격은 확장자마다 다릅니다.

## Cloudflare에서 도메인 구매하기

**1단계** [Cloudflare 대시보드](https://dash.cloudflare.com/)에 로그인합니다.

![2026031043158.webp](https://img.aikorea24.kr/images/바이브코딩-워크쓰루-1-3/26d2962ea1d82f6aee70cfde0c3636f1ed0fa292.webp)

**2단계** 왼쪽 메뉴에서 **Domain → Buy a Domain** 를 클릭합니다.

**3단계** 원하는 도메인 이름을 검색합니다. 사용 가능한 도메인과 가격 목록이 표시됩니다. 이미 누군가 사용 중인 도메인은 구매할 수 없으므로, 여러 이름을 시도해보시기 바랍니다.

**4단계** 마음에 드는 도메인을 선택하고 구매를 진행합니다. 결제 정보와 등록자 정보를 입력합니다. Cloudflare는 WHOIS 개인정보 보호를 무료로 제공하므로 등록자 정보가 외부에 공개되지 않습니다.

이 부분을 서비스하면서 매달 200~300원을 요구하는 업체도 있습니다.

**5단계** 결제가 완료되면 도메인이 Cloudflare 대시보드에 추가됩니다. DNS 설정은 자동으로 Cloudflare의 네임서버를 사용하도록 되어 있습니다.

## 도메인을 Cloudflare Pages에 연결하기

![2026031043525.webp](https://img.aikorea24.kr/images/바이브코딩-워크쓰루-1-3/ef2778f3133092794d45da9c14072d3fe5e509fa.webp)

도메인을 구매했으니 배포된 웹사이트와 연결합니다.

**6단계** Cloudflare 대시보드에서 **Workers & Pages** 를 클릭합니다. 1-2에서 만든 `my-first-website` 프로젝트를 클릭합니다.

**7단계** **Custom domains** 탭을 클릭합니다. **Set up a custom domain** 버튼을 클릭합니다.

**8단계** 구매한 도메인 주소를 입력합니다. 예를 들어 `honggildong.com`을 입력합니다. `www.honggildong.com`도 함께 설정하려면 별도로 추가합니다.

**9단계** Cloudflare에서 구매한 도메인이라면 DNS 레코드가 자동으로 설정됩니다. **Activate domain** 을 클릭합니다.

![2026031043536.webp](https://img.aikorea24.kr/images/바이브코딩-워크쓰루-1-3/f3570374b5393ec8f2d4d3892eca85b918bd317b.webp)

**10단계** 몇 분 기다리면 상태가 "Active"로 바뀝니다. 이제 웹 브라우저에 구매한 도메인 주소를 입력하면 자기소개 페이지가 나타납니다.

HTTPS 인증서도 Cloudflare가 자동으로 발급하고 관리합니다. 주소창에 자물쇠 아이콘이 표시되면 보안 접속이 적용된 것입니다. 별도로 인증서를 구매하거나 설정할 필요가 전혀 없습니다.

도메인 연결 후 몇 가지 확인할 것이 있습니다.

**기존 pages.dev 주소도 계속 작동합니다.** `my-first-website.pages.dev`와 `honggildong.com` 두 주소 모두 같은 웹사이트를 보여줍니다. 기존 주소를 누군가에게 공유했다면 걱정하지 않아도 됩니다.

**DNS 전파에 시간이 걸릴 수 있습니다.** Cloudflare에서 도메인을 구매한 경우에는 보통 몇 분이면 되지만, 다른 업체에서 구매한 도메인을 연결하는 경우 최대 24시간까지 걸릴 수 있습니다. DNS 전파란 "이 도메인은 이 서버를 가리킨다"는 정보가 전 세계 DNS 서버에 퍼지는 과정입니다.

**갱신 날짜를 기억해두세요.** 도메인은 보통 1년 단위입니다. Cloudflare는 자동 갱신 옵션을 제공하므로, 설정해두면 갱신을 깜빡해서 도메인을 잃어버리는 일을 방지할 수 있습니다.

## 시리즈1을 마치며

시리즈1 "내 첫 웹사이트" 전체를 돌아보겠습니다.

1-1에서 AI에게 대화로 요청해서 자기소개 웹페이지를 만들었습니다. 코드를 한 줄도 직접 작성하지 않았습니다. 1-2에서 그 코드를 GitHub에 올리고 Cloudflare Pages로 배포해서 인터넷에 공개했습니다. 1-3에서 나만의 도메인을 연결해서 `honggildong.com` 같은 주소를 가지게 되었습니다.

세 편의 글로 **아이디어 → 제작 → 배포 → 도메인** 까지의 전체 과정을 경험했습니다. 이것이 바이브코딩으로 무언가를 만들어 세상에 내놓는 기본 흐름입니다. 앞으로 무엇을 만들든 이 흐름은 동일합니다. 다만 만드는 것의 복잡도가 올라갈 뿐입니다.

시리즈2에서는 정적인 자기소개 페이지를 넘어, **외부 데이터를 가져와서 보여주는 서비스** 를 만들어봅니다. 공공데이터포털의 API를 활용해서 실시간 데이터를 표시하는 웹사이트를 만들 것입니다. "AI에게 코드를 받아서 파일에 붙여넣는" 수준에서 한 단계 올라가는 경험이 될 것입니다.
