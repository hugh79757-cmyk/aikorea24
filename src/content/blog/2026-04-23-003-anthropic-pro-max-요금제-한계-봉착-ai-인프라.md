Anthropic이 신규 가입자 중 일부를 대상으로 Pro 플랜에서 Claude Code를 제외하는 테스트를 진행했다가, 공개 비판이 이어지자 수 시간 만에 철회했습니다. 단순한 요금제 실험처럼 보이지만, 이 사건의 배경에는 전 AI 산업을 압박하고 있는 컴퓨팅 자원 부족이라는 구조적 문제가 있습니다.

Anthropic 성장 담당 임원은 이번 해프닝을 해명하면서 "현재 요금제는 이런 사용 패턴을 위해 설계된 것이 아니다"라고 공개 시인했습니다. 요금제 변경 시도보다 이 발언이 훨씬 더 중요한 신호입니다.

오늘의 AI 뉴스 전체 브리핑은 [aikorea24.kr 2026년 4월 23일 브리핑](https://aikorea24.kr/briefing/2026-04-23/)에서 확인하실 수 있습니다.

## 무슨 일이 있었나

미디어 분석가 에드 지트론(Ed Zitron)이 Anthropic 공식 웹사이트의 가격 페이지가 변경된 것을 포착했습니다. 모바일과 데스크톱 모두에서 Pro 플랜 항목에 Claude Code가 X 표시로 제외되어 있었고, Max 5x와 Max 20x 플랜만 Claude Code 접근을 유지하고 있었습니다. 지원 문서도 같은 방향으로 수정됐습니다.

공개 비판이 빠르게 확산되자 Anthropic 성장 담당 임원 아몰 아바사레(Amol Avasare)가 X에 해명 게시물을 올렸습니다. "신규 프로슈머 가입자의 약 2%를 대상으로 한 소규모 테스트"였으며, 기존 Pro·Max 구독자는 영향을 받지 않는다는 설명이었습니다.

그러나 지트론은 이 설명이 실제 변화와 맞지 않는다고 지적했습니다. 2%에게만 보이는 테스트라면 공개 가격 페이지와 지원 문서 전체를 바꿀 이유가 없기 때문입니다. Anthropic은 결국 변경 사항을 전면 되돌렸고, 아바사레는 혼란에 대해 공개적으로 사과했습니다.

## 임원의 발언이 더 중요한 이유

이번 해프닝에서 핵심은 요금제 변경 시도 자체가 아닙니다. 아바사레가 해명 과정에서 남긴 발언입니다.

그는 Anthropic이 "작은 조정들(주간 사용 한도, 피크 시간대 제한)을 해왔지만 사용 패턴이 많이 달라졌고, **현재 요금제는 이런 사용을 위해 만들어진 것이 아니었다**"고 밝혔습니다. Max 플랜은 Claude Code와 Claude Cowork가 존재하기 이전, 무거운 채팅 사용을 위해 설계됐다는 것입니다.

이 발언은 Anthropic이 현재의 구독 가격과 실제 사용 비용 사이에 상당한 불일치가 생겼음을 인정한 것입니다. 기업 고객 대상으로는 이미 정액제에서 토큰당 과금 방식으로 전환하는 움직임도 진행 중인 것으로 알려졌습니다.

## AI 산업 전체의 컴퓨팅 위기

Anthropic만의 문제가 아닙니다. 같은 시기에 유사한 상황이 업계 전반에서 동시에 진행됐습니다.

OpenAI는 Sora(영상 생성 AI 서비스)를 사실상 종료했습니다. 하루 100만 달러에 달하는 운영 비용을 감당하기 어렵고, 그 컴퓨팅 자원을 코딩·엔터프라이즈 제품으로 돌려야 했기 때문입니다. GitHub Copilot은 신규 가입을 잠정 중단하고 기존 플랜의 과금 구조를 변경했습니다.

숫자가 상황을 명확히 보여줍니다. 스팟 마켓(시장에서 즉시 구매하는 방식) GPU 시간당 가격은 오른 컴퓨트 가격 인덱스(Ornn Compute Price Index) 기준으로 48% 올랐습니다. Anthropic의 API 가동률은 최근 90일 평균 98.95%로, 기존 클라우드 산업의 표준인 99.99%와 상당한 격차가 있습니다. 이 때문에 Anthropic을 선호했던 기업 고객 일부가 안정성을 이유로 OpenAI로 이전하는 사례도 생겨나고 있습니다.

## Claude Code 사용자가 지금 알아야 할 것

현재 기존 Pro·Max 구독자의 Claude Code 접근은 변경되지 않았습니다. 그러나 아바사레의 발언을 토대로 보면, 요금제 구조 개편은 시점의 문제일 뿐 방향은 정해진 것으로 판단됩니다.

가능한 시나리오는 세 가지입니다. 첫째, Claude Code를 별도 애드온(추가 옵션)으로 분리하고 추가 요금을 부과하는 방식. 둘째, 현재 Max 5x·Max 20x 구조를 조정해 Claude Code 사용량에 따른 상위 티어를 신설하는 방식. 셋째, API 토큰당 과금으로 전환하는 방식.

어떤 방향으로 가든 현재의 월정액 구조에서 Claude Code를 집중적으로 사용하는 개발자·기업이 지금보다 높은 비용을 부담하게 될 가능성이 높습니다. Claude Code를 핵심 업무 도구로 사용하고 있다면, 대체 옵션과 비용 구조를 미리 검토해두는 것이 합리적입니다.

## 결론

이번 해프닝은 AI 서비스가 구독 경제(정액 요금제)와 실제 컴퓨팅 비용 구조 사이의 모순을 직면하는 과정의 한 장면입니다. AI 모델이 강력해질수록 사용당 비용도 함께 올라가는데, 고정 월정액으로 무제한에 가까운 사용을 약속하는 구조는 장기적으로 유지되기 어렵습니다. GPU 자원의 희소성이 실제 서비스 품질과 요금 구조 모두에 영향을 미치는 시대가 시작됐습니다.
